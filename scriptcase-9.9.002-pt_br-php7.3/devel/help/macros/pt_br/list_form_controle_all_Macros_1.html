sc_begin_trans - Esta macro inicializa um conjunto de transações na base de dados.

sc_change_connection - Esta macro troca dinamicamente as conexões das aplicações.

sc_commit_trans - Faz com que todas as modificações de dados realizadas desde o inicio da transação sejam parte permanente do banco de dados.

sc_concat - Esta macro é usada para combinar duas ou mais strings e/ou campos da tabela.

sc_connection_edit - Esta macro edita uma conexão existente em tempo de execução.

sc_connection_new - Esta macro permite a criação de novas conexões dinamicamente.

sc_error_continue -

Esta macro desativa o tratamento de erros de banco de dados, padrão do Scriptcase.



sc_error_delete - Esta macro configura a variável que contem a mensagem de erro do banco de dados que pode ocorrer durante a exclusão de um registro.

sc_error_insert - Esta macro configura a variável que contem a mensagem de erro do banco de dados que pode ocorrer durante a inclusão de um registro.

sc_error_update - Esta macro configura a variável que contem a mensagem de erro do banco de dados que pode ocorrer durante a atualização de um registro.

sc_exec_sql - Esta macro executa o comando SQL passado como parâmetro ou o comando SQL contido no campo tipo ação SQL informado.

sc_lookup - Esta macro executa o comando SELECT informado no segundo parâmetro e retorna os dados em uma variável.

sc_reset_change_connection - Esta macro apaga as trocas feitas usando "sc_change_connection".

sc_reset_connection_edit - Esta macro desfaz as edições de conexão feitas pela macro "sc_connection_edit".

sc_reset_connection_new - Esta macro desfaz as conexões feitas pela macro "sc_connection_new".

sc_rollback_trans - Esta macro descarta um set de transações na base de dados.

sc_select - Esta macro executa o comando SELECT informado no segundo parâmetro e retorna o dataset em uma variável.

sc_select_field - Esta macro altera dinamicamente o campo que será recuperado pela consulta.

sc_select_order - Esta macro altera dinamicamente o campo da cláusula "ORDER BY" da consulta.

sc_select_where - Esta macro adiciona dinamicamente uma condição à cláusula WHERE da consulta.

sc_set_fetchmode - Esta macro permite modificar o tipo de retorno do dataset dos comandos select.

sc_sql_injection - Esta macro protege o campo/variável contra "SQL injection"

sc_sql_protect - Essa macro protege valor digitado de acordo com o banco de dados utilizado.

sc_where_current - Esta macro disponibiliza o conteúdo do select original mais o filtro.

sc_where_orig - Esta macro disponibiliza o conteúdo da cláusula where do select original da aplicação.

Variables - Database - Variáveis especiais que contêm os dados para acesso à base de dados.
