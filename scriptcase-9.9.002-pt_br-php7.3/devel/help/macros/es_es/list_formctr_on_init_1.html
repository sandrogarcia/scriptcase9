sc_begin_trans - Esta macro inicia un conjunto de transacciones en la base de datos.

sc_change_connection - Esta macro cambia dinámicamente las conexiones de la aplicación.

sc_commit_trans - Hace que todas las modificaciones de datos realizadas desde el comienzo de la transacción sean parte permanente de la base de datos.

sc_concat - Esta macro se usa para combinar dos o más cadenas y/o campos de la tabla.

sc_connection_edit - Esta macro edita una conexión existente en tiempo de ejecución.

sc_connection_new - Esta macro crea nuevas conexiones dinámicamente.

sc_error_continue -

Esta macro desactiva el mensaje de tratamiento de error de base de datos estándar de Scriptcase para un evento.



sc_exec_sql - Esta macro ejecuta comandos SQL pasados como parámetro o un comando SQL en el tipo de acción de campo SQL.

sc_lookup - Esta macro ejecuta un comando SELECT almacenado en el segundo parámetro y devuelve los datos en una variable.

sc_reset_change_connection - Esta macro borra los cambios realizados usando "sc_change_connection".

sc_reset_connection_edit - Esta macro deshace las ediciones de conexión realizadas por la macro "sc_connection_edit".

sc_reset_connection_new - Esta macro deshace las conexiones hechas por la macro "sc_connection_new".

sc_select - Esta macro ejecuta los comandos pasados en el segundo parámetro y devuelve el conjunto de datos en una variable.

sc_set_fetchmode - Esta macro permite cambiar el tipo de retorno del conjunto de datos de los comandos seleccionados

sc_sql_injection - Esta macro se usa para proteger el campo / variable contra "inyección SQL".
